LogStateChange(string, string, string)

Logs a state change event with automatic emoji based on the state keyword.

void LogStateChange(string fromState, string toState, string reason = "")

Parameters

fromState string

The previous state (e.g., "idle", "position_open").

toState string

The new state (use keywords from remarks above for emoji).

reason string

The reason for the state change (optional, max ~100 chars recommended).

Remarks

GREEN emoji (🟢) - Active/Running states:

  • "active"Bot is active
  • "idle"Bot is idle and ready
  • "open"Position is open
  • "started"Bot/process started
  • "running"Bot is running
  • "position_open"Position opened

RED emoji (🔴) - Inactive/Error states:

  • "closed"Position/session closed
  • "stopped"Bot stopped
  • "ended"Process ended
  • "error"Error occurred
  • "failed"Action failed
  • "blocked"Bot blocked/paused

YELLOW/SPINNER emoji (🔄) - Pending/Transitional states:

  • "pending"State pending
  • "waiting"Waiting for something
  • "init"Initialization
  • "recovery"Recovery process
  • "entry_pending"Waiting for entry order fill
  • "reversing_pending"Reversing position pending

DEFAULT emoji (➡️) - Unknown/other states (not in list above)

Note: Use lowercase state names for consistent emoji mapping.
